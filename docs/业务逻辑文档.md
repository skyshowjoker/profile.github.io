## 一、产品概述

这是一款基于 **虎扑体育** 平台的 NBA 球员选秀模拟 H5 小游戏。玩家通过老虎机（Slot Machine）机制随机抽取 NBA 球队与赛季组合，从对应球员池中挑选球员组建 5 人阵容，最终由模拟引擎预测该阵容在 82 场常规赛中的战绩。核心卖点是通过随机性与策略选择，挑战 82-0 的完美赛季。

### 核心玩法一句话

> 随机球队、随机年份、五次机会，挑选出五名球员组成阵容，谁能创造出 82-0 的完美阵容？

### 3.2 详细步骤

#### Step 1: 主菜单 (screen-menu)

- 展示游戏标题「82-0」、游戏标语
- 三个入口按钮:
  - **▶ 开始游戏** — 进入游戏流程
  - **📖 玩法说明** — 弹窗展示规则
  - **👥 查看其他JRs的成绩** — 跳转虎扑论坛话题标签 (`tagId=148405`)

#### Step 2: 老虎机抽取 (spinSlot)

- 点击「🎰 抽取」按钮，老虎机开始转动
- **两个滚轮**: 球队(左) + 年份(右)
- 动画: 22 次快速切换，逐渐减速，最终锁定结果
- 结果随机从 **未使用过的 (赛季, 球队) 组合** 中选取
- 赛季范围: 2000-01 至 2024-25（共25个赛季）
- 抽取完成后，可使用跳过机会:
  - **🔄 更换球队** — 同一赛季重新抽球队（1次/局）
  - **🔄 更换年份** — 同一球队重新抽赛季（1次/局）

#### Step 3: 球员选择 (selectPlayer)

- 根据抽取的 (球队, 赛季) 组合，展示可选球员列表
- **球员筛选逻辑**:
  - "全部" 标签: 取该组合下 **Top 15（按 PTS+REB+AST 排序）+ Bottom 5**，共最多 20 人
  - "后卫" 标签: 仅显示 PG/SG 球员
  - "前锋" 标签: 仅显示 SF/PF 球员
  - "中锋" 标签: 仅显示 C 球员
  - 已选入阵容的球员不可再选
- **球员卡片显示**: 姓名、位置、薪资（$XX.XM）、球队·赛季
- **选人流程**:
  1. 点击球员卡片 → 高亮该卡片
  2. 阵容栏中该球员可打的**空位**闪烁高亮
  3. 点击闪烁的位置槽 → 确认选入该位置
  4. 若球员所有可打位置已满，提示"位置已满"
- **位置调整**:
  - 点击阵容中已填入的球员 → 进入移动模式
  - 该球员其他可打的空位闪烁
  - 点击目标位置 → 完成位置移动

#### Step 4: 循环 5 轮

- 每轮选入 1 名球员，共 5 轮
- 5 个位置: PG, SG, SF, PF, C
- 阵容栏布局: 上排(PG, SG) + 下排(SF, C, PF)

#### Step 5: 模拟结算 (runSimulation)

5 轮结束后自动触发，计算预测战绩。

## 四、模拟引擎详解

### 4.1 评分体系

#### 4.1.1 位置权重 (POSITION_WEIGHTS)

| 位置 | 得分(pts) | 篮板(reb) | 助攻(ast) | 抢断(stl) | 盖帽(blk) |
|------|-----------|-----------|-----------|-----------|-----------|
| PG   | 0.40      | 0.10      | 0.35      | 0.10      | 0.05      |
| SG   | 0.45      | 0.10      | 0.20      | 0.20      | 0.05      |
| SF   | 0.45      | 0.15      | 0.20      | 0.15      | 0.05      |
| PF   | 0.40      | 0.30      | 0.10      | 0.10      | 0.10      |
| C    | 0.40      | 0.35      | 0.10      | 0.05      | 0.10      |

#### 4.1.2 赛季基准值 (SEASON_BENCHMARKS)

动态从 NBA_DATA 计算，每个赛季一个基准值，公式为该赛季所有球员的场均统计均值:

```javascript
SEASON_BENCHMARKS[season] = {
  pts: 平均场均得分,
  reb: 平均场均篮板,
  ast: 平均场均助攻,
  stl: 平均场均抢断,
  blk: 平均场均盖帽
}
```

由于 2000s 后数据完整，所有赛季的 stl/blk 均有数据，无需像旧版年代模式那样处理缺失值。

#### 4.1.3 单球员评分算法 (playerRating)

```
对每个统计项 k (pts/reb/ast/stl/blk):
  ratio = player[k] / benchmark[season][k]
  if ratio > 1: ratio = ratio ^ 1.25   (超基准加速提升)
  n += weight[k] × ratio

base = weak ? 40 + 20n : 60 + 40n      (弱球员基准更低)
versatility = (positions数 - 1) × 3     (多位置加成)
intangibles = 是否传奇球员 ? 2.5 : 0    (无形资产加成)

playerRating = min(100, round((base + versatility + intangibles) × 10) / 10)
```

**特殊处理**:
- **弱球员** (Bottom 5): 基准分从 40 起步（正常为 60），同样 n 值下评分大幅降低
- **传奇球员** (INTANGIBLES 集合): 20 位历史级球星获得 +2.5 额外加分
- **薪资不影响评分**: 薪资仅作为显示信息，不参与模拟引擎计算

### 4.2 战绩计算

```
1. 计算 5 名球员的 playerRating
2. 几何平均: geoMean = (r1 × r2 × r3 × r4 × r5) ^ (1/5)
3. 团队总分: teamOvr = round(geoMean × 1.1 × 10) / 10
4. 胜场公式: wins = round(82 × min(teamOvr / 110, 1) ^ 2.2)
5. 负场: losses = 82 - wins
```

**非线性曲线说明**: 使用 2.2 次幂使越接近 82 胜越难达成，只有五名球员评分都极高才能接近 82-0。

**特殊修正规则**:
- 64-18 → 修正为 65-17
- 18-64 → 修正为 17-65
- 54-28 → 修正为 55-27
- 28-54 → 修正为 27-55

### 4.3 等级评定 (TEAM_GRADE_BANDS)

| 最低胜场 | 等级 | 标签 | 颜色 |
|----------|------|------|------|
| 80       | S    | 完美赛季 | 紫色 #a855f7 |
| 72       | A+   | 历史级强队 | 绿色 #22c55e |
| 62       | A    | 王朝球队 | 绿色 #22c55e |
| 57       | B    | 有力竞争者 | 蓝色 #3b82f6 |
| 50       | C    | 季后赛球队 | 金色 #f59e0b |
| 40       | D    | 乐透球队 | 灰色 #64748b |
| 0        | F    | 摆烂大军 | 红色 #ef4444 |

**80+ 胜场触发彩带庆祝动画 (confetti)**

### 5.5 薪资估算

薪资基于 NBA 薪资帽 + 球员百分位排名估算:

- 数据源: NBA 公开薪资帽数据 (2000-01: $35.5M → 2024-25: $140.6M)
- 排名指标: 综合评分 = PTS×0.45 + REB×0.2 + AST×0.2 + STL×0.08 + BLK×0.07
- 薪资档位:
  - Top 1%: cap×0.32 (顶薪)
  - Top 5%: cap×0.25
  - Top 10%: cap×0.20
  - Top 25%: cap×0.12
  - Top 50%: cap×0.07
  - 其他: cap×0.03 (底薪)
- 加 ±15% 随机浮动

## 六、关键业务规则

| 规则 | 说明 |
|------|------|
| 总轮数 | 5 轮 |
| 位置数 | 5 个 (PG, SG, SF, PF, C) |
| 可选赛季 | 2000-01 至 2024-25 (共25个赛季) |

## 七、数据管道 (fetch-nba-data.py)

### 7.1 运行方式

```bash
pip install nba_api pandas
python fetch-nba-data.py
```

### 7.2 数据获取流程

1. 从 `name_map.json` / `pos_map.json` 加载旧数据的中文/位置映射
2. 对每个赛季调用 `LeagueDashPlayerStats(season, PerGame)` 获取场均数据
3. 过滤 GP < 10 的球员
4. 从旧数据匹配中文名 (baseSlug → cname)，覆盖 ~90%
5. 未匹配的球员使用基于统计的位置推断
6. 基于薪资帽+百分位估算薪资
7. 输出 `nba-data.js`

### 7.3 位置推断

- **优先**: 从旧 nba-data.js 的 baseSlug 匹配
- **回退**: 基于统计数据的启发式推断 (高篮板+盖帽→C, 高助攻→PG, 等)
